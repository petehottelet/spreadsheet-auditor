# Benchmark Methodology

Spreadsheet Auditor's effectiveness is measured against an explicit catalog of
**seeded defects** rather than vague "found N bugs in random spreadsheets"
claims. This document explains how the benchmark is constructed, what it does
and does not measure, and how to interpret the published matrix.

## Goals

1. Give reviewers a clear answer to "what does this tool detect?"
2. Provide a regression net so silent detection drops are caught in CI.
3. Be honest about misses, value-dependent gaps, and potential false positives.

## Inputs

- **Seeded catalog**: [`benchmarks/seeded_defects.json`](../benchmarks/seeded_defects.json).
  Each entry is a `(workbook, location, rule_id, mode, requires_recalc, notes)`
  tuple.
- **Workbooks**: small, hand-built workbooks where every cell exists for a
  documented reason:
  - [`assets/test_corpus/seeded_defects/seeded-defects.xlsx`](../assets/test_corpus/seeded_defects/seeded-defects.xlsx) - regression corpus that exercises every advertised rule.
  - [`examples/demo_bad_budget.xlsx`](../examples/demo_bad_budget.xlsx) - user-facing demo.

We intentionally keep workbooks small (<50 cells) so each finding is easy to
inspect manually.

## How the benchmark runs

1. `python benchmarks/run_benchmark.py` invokes `python -m spreadsheet_auditor`
   on each workbook with `--json -` (JSON to stdout).
2. For every seed entry, the benchmark checks whether the audited findings
   contain a finding with the same `rule_id` at the same normalized location.
3. The result of each seed is one of:
   - `detected` - the auditor produced the expected finding.
   - `MISSED` - the auditor did not produce the expected finding and the check
     does not require recalculation (or recalculation was available).
   - `n/a (no recalc)` - the check requires LibreOffice recalculation, and
     recalculation was unavailable in this run. This is reported but does not
     count as a miss.
4. Any audit finding that does not match a seed is reported as an
   **unexpected finding** (potential false positive) so reviewers can confirm.

The matrix lands at [`benchmarks/seeded_defects_matrix.md`](../benchmarks/seeded_defects_matrix.md)
and is regenerated by CI.

## What this benchmark does **not** measure

- Real-world precision/recall across the long tail of spreadsheet defects in
  the wild. The seeded catalog is a fixed, curated set.
- The semantic correctness of formulas (the auditor performs structural and
  reconciliation checks, not symbolic computation).
- Sheet-style or formatting issues outside the catalog (legibility, named
  ranges, conditional formatting, charts, etc.).
- Performance on very large workbooks. Performance guardrails are exercised in
  the unit test suite, not in this benchmark.

## Reproducing locally

```bash
pip install -e .[all]
sudo apt-get install -y libreoffice-calc  # optional, enables value-dependent checks
python benchmarks/run_benchmark.py
cat benchmarks/seeded_defects_matrix.md
```

To fail the run when any seeded defect is missed or unexpected, add `--strict`:

```bash
python benchmarks/run_benchmark.py --strict
```

## Extending the catalog

When adding a new check:

1. Add a deterministic example to the seeded corpus generator
   ([`scripts/create_seeded_corpus.py`](../scripts/create_seeded_corpus.py)) or
   to the demo generator ([`examples/make_demo.py`](../examples/make_demo.py)).
2. Add a matching entry to [`benchmarks/seeded_defects.json`](../benchmarks/seeded_defects.json).
3. Run `python scripts/create_seeded_corpus.py && python examples/make_demo.py`
   to refresh artifacts, then `python benchmarks/run_benchmark.py` to update
   the matrix.
4. Confirm the new row is `detected` (or honestly labelled as a known miss in
   the notes).

## Known limitations of the benchmark

- The matrix is currently single-environment. Differences between Excel and
  LibreOffice rounding/evaluation may shift value-dependent results across
  runners.
- Detection at a *normalized* location string compares cell-level matches but
  does not assert the wording of evidence or suggested fixes.
- Sample sizes are small. Reviewers should not extrapolate the matrix to
  workbook complexity outside the catalog.
